Common Interview Questions & Concepts:

Linked List bhi data store karne ka ek tareeka hai, bilkul Array ki tarah. Lekin Array se alag, ismein elements memory mein ek saath (contiguous) store nahi hote. Balki, har element ek 'Node' hota hai.

Har Node ke do hisse hote hain:

  • Data: Asal value jo store karni hai (jaise number, string, etc.).
  • Next Pointer (Address): Agle Node ka memory address. Aakhri Node ka Next Pointer 'Null' (ya None) hota hai, jo batata hai ki list yahan khatam ho gayi.

Imagine karo ek treasure hunt jaisa, jahan har clue (Node) aapko batata hai ki agla clue kahan milega. Pehla Node 'Head' kehlata hai, jahan se list shuru hoti hai.

Array se Fark: Array ka size pehle se fix ho sakta hai aur elements memory mein line se hote hain. Linked List ka size aasani se badhaya ya ghataya ja sakta hai (dynamic size) aur elements memory mein kahin bhi ho sakte hain, bas 'Next' pointer se jude rehte hain. Isliye Linked List mein beech mein element daalna ya nikalna (agar Node ka address pata ho) Array ke mukable fast hota hai (O(1)), kyonki shifting nahi karni padti. Lekin kisi specific index wale element tak pahunchna Array mein fast (O(1)) hai, jabki Linked List mein shuru (Head) se traverse karna padta hai (O(n)).

Upar diagram mein ek Singly Linked List dikhayi gayi hai. 'Head' pointer sabse pehle Node (jisme data 30 hai) ko point kar raha hai. Har Node mein data ke saath agle Node ka address (arrow se dikhaya gaya) store hai. Aakhri Node (data 10) ka 'Next' pointer NULL hai, matlab list yahan khatam.

Mainly teen type ki hoti hain:

  1. Singly Linked List: Sabse basic type. Har Node mein sirf agle Node ka address hota hai ('Next' pointer). Hum sirf aage (forward) traverse kar sakte hain.
  2. Doubly Linked List: Har Node mein do pointers hote hain - ek agle Node ka ('Next') aur ek pichle Node ka ('Previous'). Ismein hum aage aur peeche dono taraf traverse kar sakte hain. Thodi extra memory lagti hai 'Previous' pointer ke liye.
  3. Circular Linked List: Ismein aakhri Node ka 'Next' pointer Null nahi hota, balki woh wapas pehle Node (Head) ko point karta hai. Yeh ek circle bana leti hai. Yeh Singly ya Doubly dono tarah ki ho sakti hai.

Pehle humein ek 'Node' class banani padegi, jisme data aur next pointer hoga. Fir ek 'LinkedList' class bana sakte hain jo Nodes ko manage karegi. 


# Step 1: Node Class banana
class Node:
    def __init__(self, data):
        self.data = data  # Node mein store hone wala data
        self.next = None  # Shuru mein next node ka address None hoga

# Step 2: LinkedList Class banana
class LinkedList:
    def __init__(self):
        self.head = None # Shuru mein list khali hai, toh head None hai

    # List mein elements print karne ka function
    def print_list(self):
        temp = self.head
        print("List: ", end="")
        while temp: # Jab tak temp None na ho jaye
            print(temp.data, "->", end=" ")
            temp = temp.next # Agle node pe jao
        print("None") # List ke end mein None dikhao

    # Shuru mein Node add karne ka function
    def add_at_beginning(self, new_data):
        new_node = Node(new_data) # Naya node banaya
        new_node.next = self.head # Naye node ka next abhi current head banega
        self.head = new_node      # Ab naya node hi head ban gaya

# --- Example Usage ---
my_list = LinkedList()

# Kuch elements add karte hain (shuru mein)
my_list.add_at_beginning(10) # List: 10 -> None
my_list.add_at_beginning(20) # List: 20 -> 10 -> None
my_list.add_at_beginning(30) # List: 30 -> 20 -> 10 -> None

# List ko print karke dekhte hain
my_list.print_list() # Output: List: 30 -> 20 -> 10 -> None

Yeh ek bahut hi basic example hai. Ek complete Linked List implementation mein aur bhi functions hote hain jaise end mein add karna, beech mein add karna, delete karna, search karna etc.

  • Access (k-th element): O(n) - Shuru se traverse karna padta hai.
  • Search: O(n) - Worst case mein poori list check karni pad sakti hai.
  • Insertion (Beginning mein): O(1) - Sirf head pointer ko update karna hota hai. Bahut fast.
  • Insertion (End mein): O(n) - Aakhri node tak traverse karna padta hai (agar tail pointer maintain na kiya ho toh). Agar tail pointer maintain karte hain toh O(1).
  • Insertion (Beech mein - agar node ka pointer pata ho): O(1) - Connections update karna fast hai. Agar index diya ho toh pehle us node tak pahunchna padega (O(n)).
  • Deletion (Beginning mein): O(1) - Head pointer update karna hai.
  • Deletion (End mein): O(n) - Aakhri se pehle wale node tak traverse karna padta hai.
  • Deletion (Beech mein - agar node ka pointer pata ho): O(1) - Connections update karna fast hai. Agar value/index diya ho toh pehle us node tak pahunchna padega (O(n)).

(n = list mein kitne nodes hain)

Space Complexity: O(n) - Jitne elements utni memory.

Key Differences:

Feature Array Linked List
Memory Allocation Contiguous (Ek saath) Non-contiguous (Kahin bhi)
Size Fixed (Static Array) / Dynamic but resizing costly Dynamic (Easy to grow/shrink)
Access Time (by Index) O(1) O(n)
Insertion/Deletion (Beginning) O(n) (Shifting needed) O(1)
Insertion/Deletion (Middle) O(n) (Shifting needed) O(1) (if pointer given), else O(n) to find
Insertion/Deletion (End) O(1) (amortized for dynamic array) O(n) (or O(1) if tail pointer used)
Extra Memory Nahi (for data itself) Haan (for 'Next' pointers)

Kab kya use karein?

  • Agar aapko baar baar random elements ko unke index se access karna hai, toh Array behtar hai.
  • Agar aapko baar baar list ke shuru ya beech mein elements insert/delete karne hain aur size ka pehle se andaaza nahi hai, toh Linked List accha option hai.
